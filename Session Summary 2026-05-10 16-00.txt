Session Summary — 2026-05-10 16:00

Issues Diagnosed & Fixed
========================

1. Google Sheets OAuth Token Expiry (invalid_grant)
   Root cause: OAuth consent screen was in "Testing" mode, which expires refresh tokens after 7 days.
   Fix: Published the OAuth app (APIs & Services → OAuth consent screen → "Publish App"), then reconnected
   the "Google Sheets account" credential in n8n.
   Impact: Affected all Lead Finder workflows that write/read Google Sheets (Lead Finder, Lead Finder - Enrich DM,
   Lead Finder - Read, Lead Finder - Clear).

2. Google Sheets 404 NOT_FOUND ("Read Sheet" / "Read Existing Leads1")
   Root cause: Cascading symptom of the OAuth failure above. When the token was invalid, the Google Sheets API
   returned 404 during the sheet-lookup step rather than a clean 401.
   Fix: Resolved automatically once the OAuth token was renewed.
   Note: The spreadsheet (1ZP3TWy5kzvkKegxTEwzoVigOHFFBXC5tIsIh1KFFuzg) still exists with the correct
   tab name "Sheet1" — no node config changes were needed.

3. Google Places API Key Expired (API_KEY_INVALID)
   Root cause: The GOOGLE_PLACES_API_KEY stored in n8n's environment variable had expired.
   How the key flows: Read Config1 (Code node) reads $env.GOOGLE_PLACES_API_KEY and passes it as the
   X-Goog-Api-Key header to all Places API HTTP Request nodes.
   Fix: Regenerate the key in Google Cloud Console → APIs & Services → Credentials, then update in:
     - /docker/n8n/docker-compose.yml on the server (restart with: cd /docker/n8n && docker compose up -d --force-recreate)
     - lead-finder/.env.local (local dev, for reference)
   Note: Service account key creation was blocked by org policy (iam.disableServiceAccountKeyCreation),
   so a service account approach was not viable.

4. Enrich DM — Vercel 500 Error
   Root cause: Two environment variables required by /api/enrich-dm were missing from Vercel's production
   environment. The other routes use N8N_WEBHOOK_URL (which was set), but enrich-dm uses different vars.
   Missing vars:
     - N8N_WEBHOOK_URL_BASE
     - ENRICH_DM_SECRET
   Fix: Add both to Vercel dashboard → lead-finder project → Settings → Environment Variables,
   then redeploy.
